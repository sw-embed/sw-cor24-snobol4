SNOBOL4 program
This version is designed to be simple and demonstrative, not maximally elegant.

* profile_match.sno
*
* Demo dating-profile matcher for SNOBOL4.
* Record format:
*   name|sex|seeking|age|height|eye|interest1,interest2,...
*
* Reads records from standard input until END
* Then prints best match for each person.

        DEFINE('HAS(LIST,ITEM)')
        DEFINE('COMMON(L1,L2)')
        DEFINE('SCORE(I,J)D,C')

        NAMES     = ARRAY('1:50')
        SEXES     = ARRAY('1:50')
        SEEKS     = ARRAY('1:50')
        AGES      = ARRAY('1:50')
        HEIGHTS   = ARRAY('1:50')
        EYES      = ARRAY('1:50')
        INTS      = ARRAY('1:50')

        N = 0

* ---------- LOAD RECORDS ----------

READ    LINE = INPUT                               :F(LOADED)
        IDENT(LINE,'')                             :S(READ)
        IDENT(LINE,'END')                          :S(LOADED)

        LINE BREAK('|') . NAME
+            '|' BREAK('|') . SEX
+            '|' BREAK('|') . SEEK
+            '|' BREAK('|') . AGE
+            '|' BREAK('|') . HEIGHT
+            '|' BREAK('|') . EYE
+            '|' REM . INTERESTS                   :F(BADREC)

        N = N + 1
        NAMES<N>   = NAME
        SEXES<N>   = SEX
        SEEKS<N>   = SEEK
        AGES<N>    = +AGE
        HEIGHTS<N> = +HEIGHT
        EYES<N>    = EYE
        INTS<N>    = ',' INTERESTS ','

        :(READ)

BADREC  OUTPUT = 'Bad record: ' LINE
        :(READ)

* ---------- FIND BEST MATCH FOR EACH PERSON ----------

LOADED  I = 1

OUTER   GT(I,N)                                   :S(DONE)
        BESTJ = 0
        BESTS = -1
        J = 1

INNER   GT(J,N)                                   :S(REPORT)
        EQ(I,J)                                   :S(NEXTJ)

        S = SCORE(I,J)                            :F(NEXTJ)
        GT(S,BESTS)                               :F(NEXTJ)
        BESTS = S
        BESTJ = J

NEXTJ   J = J + 1                                 :(INNER)

REPORT  EQ(BESTJ,0)                               :S(NOMATCH)
        OUTPUT = NAMES<I> ' -> ' NAMES<BESTJ> '  score=' BESTS
        :(NEXTI)

NOMATCH OUTPUT = NAMES<I> ' -> no compatible match'

NEXTI   I = I + 1                                 :(OUTER)

DONE    END

* ---------- FUNCTIONS ----------

* HAS(LIST,ITEM)
* succeeds if LIST contains ITEM as a whole comma-delimited token
HAS     LIST ',' ITEM ','                         :S(RETURN)
                                                :(FRETURN)

* COMMON(L1,L2)
* count shared interests from a small fixed vocabulary
COMMON  COMMON = 0

        HAS(L1,'music')                           :F(C1)
        HAS(L2,'music')                           :F(C1)
        COMMON = COMMON + 1
C1      HAS(L1,'books')                           :F(C2)
        HAS(L2,'books')                           :F(C2)
        COMMON = COMMON + 1
C2      HAS(L1,'hiking')                          :F(C3)
        HAS(L2,'hiking')                          :F(C3)
        COMMON = COMMON + 1
C3      HAS(L1,'movies')                          :F(C4)
        HAS(L2,'movies')                          :F(C4)
        COMMON = COMMON + 1
C4      HAS(L1,'chess')                           :F(C5)
        HAS(L2,'chess')                           :F(C5)
        COMMON = COMMON + 1
C5      HAS(L1,'art')                             :F(C6)
        HAS(L2,'art')                             :F(C6)
        COMMON = COMMON + 1
C6      HAS(L1,'coding')                          :F(C7)
        HAS(L2,'coding')                          :F(C7)
        COMMON = COMMON + 1
C7      HAS(L1,'dancing')                         :F(CEND)
        HAS(L2,'dancing')                         :F(CEND)
        COMMON = COMMON + 1
CEND    :(RETURN)

* SCORE(I,J)
* fails if pair is incompatible
SCORE   DIFFER(SEXES<I>,SEEKS<J>)                 :S(FRETURN)
        DIFFER(SEXES<J>,SEEKS<I>)                 :S(FRETURN)

        SCORE = 50

* age difference
        D = ABS(AGES<I> - AGES<J>)
        LE(D,2)                                   :S(AGE20)
        LE(D,5)                                   :S(AGE10)
        LE(D,8)                                   :S(AGE5)
        :(HEIGHT)

AGE20   SCORE = SCORE + 20                        :(HEIGHT)
AGE10   SCORE = SCORE + 10                        :(HEIGHT)
AGE5    SCORE = SCORE + 5

* height difference
HEIGHT  D = ABS(HEIGHTS<I> - HEIGHTS<J>)
        LE(D,3)                                   :S(H8)
        LE(D,6)                                   :S(H4)
        :(EYE)

H8      SCORE = SCORE + 8                         :(EYE)
H4      SCORE = SCORE + 4

* eye color
EYE     IDENT(EYES<I>,EYES<J>)                    :S(EYEYES)
        :(INTERS)

EYEYES  SCORE = SCORE + 3

* shared interests
INTERS  C = COMMON(INTS<I>,INTS<J>)
        SCORE = SCORE + (C * 6)
        :(RETURN)
What this demonstrates well in SNOBOL4
This is why I think it is a nice interpreter test/demo:

1. Pattern-based parsing of records
The record parser uses BREAK('|'), literal delimiters, and REM to split a line into fields. That is a very natural SNOBOL4 use case. SNOBOL4 statements consist of subject, pattern, replacement, and transfer fields, and a pattern match can drive control flow through success/failure branches. 

2. Arrays and mixed string/integer handling
SNOBOL4 has first-class arrays and automatic data conversion in many contexts, so storing names, ages, eye colors, and interest strings in parallel arrays is straightforward. The ARRAY() primitive creates arrays, and SNOBOL4 variables can hold different data types as needed. 

3. User-defined functions with success/failure style
The helper functions HAS, COMMON, and SCORE fit the SNOBOL4 model well. Program-defined functions are created with DEFINE, return by assigning to a variable with the same name as the function, and exit via RETURN or FRETURN. 

Expected output shape

The exact winners depend on how your interpreter handles arithmetic and function success/failure, but the structure should be stable.
